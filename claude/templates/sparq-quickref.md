# SparQ Quick Reference

## Commands (in Claude Code)

| Command | What it does |
|---------|-------------|
| `/sparq:start` | Interactive guide — start here! |
| `/sparq:generate {ticket}` | Generate tests from Jira ticket |
| `/sparq:validate {path}` | Check if tests match current UI |
| `/sparq:sync {ticket}` | Update tests after requirement changes |
| `/sparq:regression {bug}` | Create regression test for a bug |
| `/sparq:export` | Push tests to TestRail, Qase, or Jira |
| `/sparq:config` | View or edit SparQ settings |

## CLI Commands (in terminal)

| Command | What it does |
|---------|-------------|
| `npx sparq-assistant init` | Set up SparQ in your project |
| `npx sparq-assistant doctor` | Diagnose installation issues |
| `npx sparq-assistant update` | Update to latest version |
| `npx sparq-assistant clean` | Remove generated artifacts |

## Test Categories

- **HP** = Happy Path (normal user flows)
- **VE** = Validation & Error (bad input, error handling)
- **SEC** = Security (auth, injection, permissions)
- **EC** = Edge Case (boundary values, race conditions)
- **A11Y** = Accessibility (keyboard, screen reader, ARIA)

## ID Formats

- `REQ-{feature}-{NNN}` — Requirement ID
- `TC-{feature}-{CAT}-{NNN}` — Test Case ID (feature-category-number)
- `REG-{ticket}-{NNN}` — Regression Test ID
- `VF-{N}` — Validation Finding ID
